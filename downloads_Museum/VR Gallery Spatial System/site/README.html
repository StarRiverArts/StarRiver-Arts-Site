VR Gallery Spatial System

虛擬實境美術館・空間設計準則

A spatial design system for VR / virtual museum and gallery environments. It is not a visual-identity or UI system — wayfinding, signage and on-screen UI are governed by a separate visual system. This system governs the space itself: materials, light, scale, whitespace, exhibit reserve volumes, and curve geometry — all calibrated for a 130 cm VR avatar with eye-height at 130 cm.

本系統規範 VR 美術館 / 畫廊的空間設計。視覺識別、平面、導覽 UI 由另一套設計系統負責；本系統處理「空間本體」：材質、光、空間尺度、留白、展品預留量體與曲線幾何，均以 130 公分高的 VR avatar 為基準，視覺高度亦為 130 公分。

Scope · 系統範圍

01

Materials × Light

材質 × 光源

Each material documented as a triplet: target → simulated actual → suggested adjustment. Default key light 5000K · CRI 95.

02

Spatial Ergonomics

空間人因

Viewing distance, viewing angle, mounting height — derived from the 130 cm avatar. Comfort cone ±30°, focus cone ±15°.

03

Whitespace as Structure

留白即結構

The principle of liú bái 留白 translated into measurable rules for inter-exhibit spacing and object grouping.

04

Exhibit Reserve Volumes

展品預留量體

Four reserve sizes — S / M / L / XL — for stress-testing layouts before any real work is placed.

05

Curve Geometry

曲線幾何

Approved radii for soft architecture: 5 / 20 / 60 / 120 / 240 cm. Alcoves, plinth corners, vault transitions.

06

Spatial Planner

空間規劃工具

Live plan-and-elevation tool. Drop reserve volumes, validate viewing cones against the 130 cm avatar.

Content Fundamentals · 語氣與書寫

Voice · 語氣

  • Declarative, dimensional, neutral. State the rule, then state the reason.
  • Third-person and imperative. Not "you should…".
  • Sentence case for prose. ALL-CAPS reserved for token names and section headers in drawings.
  • Always include units: cm, °, K, lux.
  • 中英雙語並列。文化專屬詞 (留白、立面圖) 中文先；國際技術詞 (LUX, CCT) 英文先。
  • No exclamation marks. No emoji. No "magic" / "delight" / consumer-product vocabulary.

Visual Foundations · 視覺基礎

Type · 字體

  • TASA Orbiter Deck — Latin primary. Drop licensed .woff2 into fonts/.
  • 思源黑體 (Noto Sans TC) — 中文主字。從 Google Fonts CDN 載入，OFL 授權。
  • JetBrains Mono — dimensions, tokens, drawings.
  • Sora — fallback for TASA Orbiter Deck when no licensed copy is bundled.
  • Tabular numerals on every dimension.

Documentation chrome · 文件外觀

  • Paper #FAFAF7, ink #1A1A1A, dim ochre #B47B2E.
  • Line weights 0.5 / 1 / 1.5 / 2 px — drafting convention.
  • 10 cm spatial grid. 8 px UI grid.
  • No gradients, no shadows, no photographs, no patterns.
  • 0 radius on documentation cards. Real curves live only inside the designed space.
  • 不使用 emoji。所有符號為線稿 SVG。

Iconography · 符號慣例

  • Architectural drafting symbols only — north arrow, scale bar, eye-level datum, dimension witness lines, viewer cone, curve template.
  • SVG line-glyphs at 1 px / 1.5 px stroke. No icon font, no PNG icons, no emoji.
  • 觀眾用的指引圖示由視覺識別系統負責，不在此規範範圍。
  • Symbol inventory: N arrow · scale bar · eye datum · plan-cut · elevation reference · dimension witnesses · reserve markers (S/M/L/XL) · light source · viewer cone · curve template.
  • Under 20 marks total. Lives inline in HTML, not as a font or sprite.
